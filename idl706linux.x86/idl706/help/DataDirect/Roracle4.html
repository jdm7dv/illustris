Connecting to a Data Source Using a Logon Dialog Box

Some ODBC applications display a logon dialog box when you are connecting to a data source. In these cases, the data source name has already been specified. For Oracle, the dialog box is as follows:

In this dialog box, perform the following steps:

  1. In the Server Name field, type the client connection string of the computer containing the Oracle database tables you want to access. Or, on Windows, select the string from the Server Name drop-down list, which displays the names you specified in the ODBC Oracle driver Setup dialog box.

    2. For local servers, the SQL*Net connection string has the form database_name, where database_name identifies your Oracle database.

    If the SQL*Net connection string contains semicolons, enclose it in quotation marks. See your SQL*Net documentation for more information.

    For Oracle7 and Oracle8 remote servers, the Oracle TNS Client connection string has the form, TNSNAME, where TNSNAME is the alias name of the Oracle Listener on your network.

  2. If required, type your Oracle user name.
  3. If required, type your Oracle password.
  4. Click OK to log on to the Oracle database installed on the server you specified and to update the values in the Registry.

NOTE: You can also use OS Authentication to connect to an Oracle database. See "OS Authentication" for details.
